# 手書きメモ取り込み 運用手順書（P3）

過去100品番の手書き工程メモを、工程分析表の「下書き」としてアプリに取り込むための手順です。
日常運用（新しい品番のメモ）もこの同じ手順で行います。

> **原則**：AIは「書き起こし」だけ。清書・並べ替え・時間の補完はさせない。
> 時間はメモに書いてある通りに取り込み、**最終見直しはリーダーが行う**。

---

## 全体の流れ

```
① メモを撮影 → ② AIに書き起こさせる → ③ アプリに貼り付け → ④ プレビュー確認 → ⑤ 確定 → ⑥ エディタで最終見直し
```

---

## ① メモを撮影する

- 1品番＝1メモを基本に、ページ全体が入るように撮る
- 折り目・影で文字が読めない部分は撮り直す
- 複数ページある場合は順番が分かるように撮る

## ② AIに書き起こさせる

AI（Claudeなど）のチャットに**写真を添付**し、下のプロンプトを**一字も変えずにそのままコピペ**して送る。

**↓ ここから下をコピー ↓**

```
添付した手書きの縫製工程メモを、次のルールで書き起こしてください。

- 1行1工程で「パーツ｜作業内容｜時間」の形式（区切りは全角の｜）
- メモに書いてある通りに書き起こすこと。清書・言い換え・並べ替え・補完は禁止
- 時間はメモの表記のまま書くこと（1:32 / 1'32 / 92 など）。メモに時間が書いていない工程は時間欄を空にする
- パーツ名が書かれていない行はパーツ欄を空にする（「前の行と同じ」と判断して埋めない）
- パーツ名だけが書かれた見出しの行は、その1行をパーツ名だけで出力する
- 読めない文字は「？」と書く。推測で埋めない
- 工程の行だけを出力する。前置き・説明文・表の枠は出力しない
```

**↑ ここまでコピー ↑**

AIの出力はこのような形になる（例）：

```
見頃｜脇はぎ｜1:20
｜ロック｜0:40
袖｜袖付け｜92
｜袖口三巻き｜1'10
カフス｜カフス付け｜
```

**チェックポイント**：出力の行数がメモの工程数と合っているか、ここで一度目視する。合っていなければ「◯行目が抜けている」とAIに返して出し直させる。

## ③ アプリに貼り付ける

1. 品番を開く（品番マスター → 該当品番）
2. 「📐 工程分析表」を押す
3. 「📋 メモから取り込む（書き起こしを貼り付け）」を選ぶ
4. ②の出力を貼り付ける

※「メモから取り込む」は工程表が**まだ無い**品番でだけ出ます。既に工程表がある品番は上書きできません（誤消去防止）。

## ④ プレビューで確認する

貼り付けると同時にプレビューが出ます。確認するのは3つ：

| 確認項目 | 見る場所 | 対応 |
|---|---|---|
| 工程数がメモと合っているか | 「◯工程」 | 行の抜け・くっつきがあれば貼り付けテキストを直す |
| 時間未記入の件数 | 「時間未記入 ◯件」（灰色） | メモに無いなら未記入のままでよい |
| 読めない時間 | 「読めない時間 ◯件」（赤） | 赤い行の時間をメモと見比べて直す |

パーツ欄が空の行は自動で直前のパーツを継承します（プレビューで確認できます）。

## ⑤ 確定する

「この内容で工程表の下書きを作る」を押すと工程表が作られ、そのままエディタが開きます。
保存は既存の仕組み（GAS）に自動で乗るので、特別な操作は不要です。

## ⑥ エディタで最終見直し（リーダー）

- 時間の読み違い（例：1:32 と 7:32）がないか、メモと突き合わせる
- 「？」になっている文字を直す
- 針・糸・運転などの頭書き情報を記入する

---

## 取り込み形式の仕様（詳細）

- 1行1工程：`パーツ｜作業内容｜時間`
- 区切りは 全角｜／半角|／タブ のどれでも可（行ごとに違ってもよい）
- パーツ欄が空 → 直前の行のパーツを継承
- 区切りの無い行 → パーツ見出し行として扱い、以降の行に継承される
- 時間は `1:32`／`1'32`／`92`（秒）／空欄 のどれでも可。`1'32`・全角数字は表記だけ `1:32` に直す（値は変えない）
- 空行は無視
- 4列目以降があれば備考（note）に入る（捨てない）

---

## 精度検証の記録（過去100品番の取込前に、まず3〜5品番で）

以下を記録し、問題の傾向がつかめてから残りを流す：

| 品番 | メモの工程数 | 取込後の工程数 | パーツ継承の誤り | 時間の読み違い（件数と内容） | かかった時間 |
|---|---|---|---|---|---|
| 例) A-2023-014 | 24 | 24 | なし | 1件（1:32→7:32） | 6分 |

**よくある問題と対処**：

- 行数が合わない → メモの2行が1行にくっついている。書き起こしを改行で分ける
- 「読めない時間」が多い → AIが `？` を時間欄に入れている。メモと見比べて手で直す
- パーツ継承がおかしい → メモのパーツ見出しをAIが作業内容と誤認。見出し行は「パーツ名だけの行」にする
- プレビューに何も出ない → 表以外の説明文だけ貼っている。②のプロンプトで出し直す
